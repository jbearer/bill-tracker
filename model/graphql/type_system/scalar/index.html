In GraphQL, primitives are called scalars.

Scalars make up the leaves of a graph-oriented data model. They are primitive types like integers and strings, upon which more complex object types are recursively built.

Unlike complex Resource types, scalars do not have properties or relationships to other objects. They are simple, atomic objects whose only property is their value. Users also cannot create new scalar types, so backends are able to rely on there being a known, finite set of scalar types upon which to build their own data model.
